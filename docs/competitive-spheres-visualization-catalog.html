Competitive Spheres -- Visualization Catalog

10 visualization approaches for showing how dimension values (MOA, indication, phase, company, ROA, etc.) overlap and intersect across clinical trial products. Reference document for feature planning.

A. Classic Venn (2-3 circles)

Pick 2-3 dimension values. Overlaps are geometric and intuitive. The center intersection is the "money zone" -- products matching ALL criteria. Click any region to drill in.

Instantly readable Familiar to everyone Clickable regions Max 3 dimensions 2-3 sets

B. Euler Diagram (4-5 ellipses)

Stretches Venn to 4-5 ellipses. More dimensions visible, but intersections become harder to parse and regions are irregularly shaped. Works when some pairs don't overlap.

More dimensions Still spatial Hard to read past 4 Can't size proportionally 4-5 sets

C. UpSet Plot

Bar chart where each bar is an intersection. Dot matrix below shows which sets combine. Bars sorted by size -- biggest overlaps and rarest positions are immediately visible. Scales to 10+ dimensions. Common in pharma/biotech research.

Scales massively Precise counts Shows ALL intersections ranked Familiar to pharma analysts Less visually intuitive 5-15+ sets

D. Intersection Matrix (Heatmap)

Pairwise grid. Diagonal = totals, off-diagonal = shared products. Color intensity = count. Zeros are instant white-space signals. Can mix dimensions (MOA rows, indication columns).

Scales to 10+ White space jumps out Familiar grid Mix dimension types Only pairwise overlaps 5-20 values

E. Network Graph

Nodes = dimension values (sized by count). Edges = shared products (thickness = overlap). Clusters emerge naturally -- tightly connected = saturated, isolated = differentiated. Force-directed layout.

Scales well Clusters reveal neighborhoods Visually striking Mix dimension types Non-deterministic layout Multi-way needs drill-down 5-30+ nodes

F. Chord Diagram

Arcs around a ring = dimension values (sized by total). Ribbons connect values sharing products -- thicker = more overlap. Missing ribbons = white space. Hover to isolate one arc's connections.

Beautiful and compact All pairwise at once Missing ribbons = gaps Only pairwise Ribbons can tangle 4-12 arcs

G. Sankey / Alluvial Flow

Products flow left-to-right through dimension columns (Company -> MOA -> Indication -> Phase). Ribbon width = how many products take each path. Thick flows = crowded lanes, thin = niche. User picks column order.

Full multi-dim paths Dominant vs. niche flows User controls columns Tangles with many values Hard to trace specifics 3-5 columns, 5-8 values each

H. Competitive Positioning Scatter

Each bubble = MOA+Indication pair. X = competitor count. Y = highest phase reached. Size = trial volume. Top-right = red ocean (crowded, mature). Bottom-left = blue ocean (early, uncrowded). Built-in strategic narrative.

Instant red/blue ocean read 4 dimensions at once Familiar to BD teams Fixed pairs, not arbitrary sets Bubbles can overlap 10-50 bubbles

I. Parallel Coordinates

Each vertical axis = a dimension. Each thread = one product tracing through all dimensions. Click any value to highlight all threads passing through it. Bundles of threads reveal competitive corridors.

Full product profiles Brushing filters everything Reveals corridors Cluttered at 100+ products Unfamiliar to some 5-8 axes, up to ~80 threads

J. Nested Treemap

Rectangle area = product count. Outer blocks = MOA, nested = indication, innermost = phase. Click to zoom into any block. Biggest rectangles = most crowded spaces. Tiny/missing = white space. Bloomberg-adjacent data density.

Extreme data density Area makes sizes obvious Zoom-to-drill is natural Fixed hierarchy Hard to compare non-adjacent 3 nesting levels, many values

Combination Ideas

  • Network (E) + Venn (A): Network as the overview, click a cluster to see a Venn of 2-3 nodes within it
  • Treemap (J) + UpSet (C): Treemap as the landing view, click a block to see UpSet for that subset
  • Scatter (H) + Matrix (D): Scatter for strategic positioning, matrix for detailed pairwise analysis
  • Sankey (G) + Venn (A): Sankey for the flow overview, Venn to drill into specific intersection
  • Network (E) + Matrix (D): Network for visual discovery, matrix for precise pairwise counts

Available Data Dimensions

  • Core entities: Company, Product, Trial, Therapeutic Area
  • Classification: Mechanism of Action (MOA), Route of Administration (ROA)
  • Development: Phase (PRECLIN, P1-P4, APPROVED, LAUNCHED), Recruitment Status
  • ClinicalTrials.gov: Study Type, Sponsor Type, Countries/Regions, Conditions, Intervention Type, FDA Designations, Design (allocation, masking, purpose)
  • Counting units (user-selectable): Products (default), Trials, Companies

Design Constraints

  • Pure SVG rendering (no charting libraries) -- consistent with existing timeline and bullseye views
  • Teal primary accent, slate neutrals, phase-based color progression
  • WCAG 2.1 AA compliant, keyboard navigable
  • Light mode only, data-dense, Bloomberg Terminal aesthetic
  • Angular 19 signals + Supabase RPC for data aggregation
