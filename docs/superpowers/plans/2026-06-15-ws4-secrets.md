# WS4 Secrets Management (Infisical) Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.
>
> **Note on execution style:** WS4 is mostly interactive cloud-console + CLI ops done WITH the user (Infisical dashboard, OAuth-style integrations), not autonomous code edits. Many tasks are "user does X in the dashboard, then we verify Y." Exact Infisical UI labels are confirmed live against the current product; the navigation and the values to enter are specified here. Run this inline with the user, not via background subagents.

**Goal:** Make Infisical Cloud the single canonical source of truth for every live Clint secret, syncing out to GitHub Actions + Cloudflare Worker secrets, with a weekly age-encrypted break-glass export, so secrets are centralized, auditable, and recoverable.

**Architecture:** Infisical Cloud (free tier) holds secrets in project `clint`, environments `dev`/`prod`/`shared`, folders by domain. One-way sync/push to GitHub Actions secrets (GitHub App) and Cloudflare Worker secrets (Cloudflare integration). A GitHub-OIDC read-only machine identity backs a weekly workflow that age-encrypts all secrets and uploads them to the R2 + B2 backup buckets. The laptop uses `infisical run` for tofu, retiring `.env.local`. Rotation and the offline age private key are out of scope.

**Tech Stack:** Infisical Cloud, Infisical CLI, GitHub Actions (OIDC), `age`, existing `scripts/backup/upload.sh`, OpenTofu (consumer).

**Source of truth:** `docs/superpowers/specs/2026-06-12-ws4-secrets-design.md`.

---

## Safety rules (apply to every phase)
- **Never enable a sync before the correct values are loaded.** The GitHub/Cloudflare sync OVERWRITES the target's existing secrets with whatever is in Infisical. Load and double-check an environment's secrets in Infisical first, then enable that environment's sync.
- **GitHub Actions secrets are write-only.** You cannot read current values back. Re-collect each value from its origin (Cloudflare/Supabase dashboard, the password-manager inventory) or rotate it. Never assume you can copy it out of GitHub.
- **Dev before prod.** Cut over and verify the `dev` environment fully before touching `prod`.
- **Keep fallback.** Do not delete or hand-edit the old GitHub/CF secrets until the synced value is verified by a green run.
- **age private key (`BACKUP_AGE_PRIVATE_KEY`) never enters Infisical.** It stays a direct GitHub secret + offline vault. This is the one deliberate exception.

### Task 8: Write the break-glass export script
**Files:**
- Create: `scripts/secrets/export-bundle.sh`
- Test: manual local decrypt (integration), see Task 10.

- [ ] **Step 1: Create the script**

```bash
#!/usr/bin/env bash
# Break-glass export: pull every Infisical secret (all envs, recursive over
# folders), bundle, and age-encrypt to the backup PUBLIC key so only the offline
# private key can decrypt. Prints the encrypted bundle path as the last stdout line.
#
# Requires: infisical CLI (authenticated via INFISICAL_TOKEN), age, zstd, tar.
set -euo pipefail

usage() { echo "usage: $0 --project-id <id> --recipient <age-pubkey> --outdir <dir>" >&2; exit 1; }
PROJECT_ID=""; RECIPIENT=""; OUTDIR="./_secrets"
while [ $# -gt 0 ]; do
  case "$1" in
    --project-id) PROJECT_ID="$2"; shift 2 ;;
    --recipient)  RECIPIENT="$2";  shift 2 ;;
    --outdir)     OUTDIR="$2";     shift 2 ;;
    *) usage ;;
  esac
done
[ -n "$PROJECT_ID" ] && [ -n "$RECIPIENT" ] || usage
: "${INFISICAL_TOKEN:?INFISICAL_TOKEN must be set (machine identity access token)}"

mkdir -p "$OUTDIR"
work="$(mktemp -d)"
trap 'rm -rf "$work"' EXIT

# One dotenv file per environment, recursive across all folders.
for env in shared dev prod; do
  infisical export \
    --projectId "$PROJECT_ID" \
    --env "$env" \
    --recursive \
    --format=dotenv > "$work/$env.env"
done

stamp="$(date -u +%Y%m%dT%H%M%SZ)"
bundle="$OUTDIR/secrets-$stamp.tar.zst.age"
tar -C "$work" -cf - . | zstd -q | age -r "$RECIPIENT" -o "$bundle"
echo "$bundle"
```

### Task 9: Write the break-glass workflow
**Files:**
- Create: `.github/workflows/secrets-break-glass.yml`

- [ ] **Step 1: Create the workflow**

```yaml
name: Secrets break-glass export
on:
  schedule:
    - cron: "0 8 * * 1"   # weekly, Monday 08:00 UTC
  workflow_dispatch:

permissions:
  contents: read
  id-token: write         # mint GitHub OIDC token for Infisical auth

concurrency:
  group: secrets-break-glass
  cancel-in-progress: false

jobs:
  export:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4

      - name: Install tooling (infisical, age, zstd, jq)
        run: |
          set -euo pipefail
          curl -1sLf 'https://artifacts-cli.infisical.com/setup.deb.sh' | sudo -E bash
          sudo apt-get update -qq
          sudo apt-get install -y -qq infisical age zstd jq

      - name: Authenticate to Infisical via GitHub OIDC
        id: auth
        env:
          IDENTITY_ID: ${{ secrets.INFISICAL_MACHINE_IDENTITY_ID }}
        run: |
          set -euo pipefail
          oidc="$(curl -sS -H "Authorization: bearer $ACTIONS_ID_TOKEN_REQUEST_TOKEN" \
            "$ACTIONS_ID_TOKEN_REQUEST_URL&audience=https://infisical.com" | jq -r '.value')"
          token="$(infisical login --method=oidc-auth \
            --machine-identity-id="$IDENTITY_ID" \
            --oidc-jwt="$oidc" --plain --silent)"
          echo "::add-mask::$token"
          echo "token=$token" >> "$GITHUB_OUTPUT"

      - name: Build encrypted bundle
        id: bundle
        env:
          INFISICAL_TOKEN: ${{ steps.auth.outputs.token }}
        run: |
          set -euo pipefail
          out="$(scripts/secrets/export-bundle.sh \
            --project-id '${{ secrets.INFISICAL_PROJECT_ID }}' \
            --recipient '${{ secrets.BACKUP_AGE_PUBLIC_KEY }}' \
            --outdir ./_secrets | tail -1)"
          echo "bundle=$out" >> "$GITHUB_OUTPUT"

      - name: Upload to R2 + B2 (secrets/ prefix)
        env:
          R2_BACKUP_BUCKET: ${{ secrets.R2_BACKUP_BUCKET }}
          R2_S3_ENDPOINT: https://${{ secrets.CLOUDFLARE_ACCOUNT_ID }}.r2.cloudflarestorage.com
          R2_BACKUP_ACCESS_KEY_ID: ${{ secrets.R2_BACKUP_ACCESS_KEY_ID }}
          R2_BACKUP_SECRET_ACCESS_KEY: ${{ secrets.R2_BACKUP_SECRET_ACCESS_KEY }}
          B2_BACKUP_BUCKET: ${{ secrets.B2_BACKUP_BUCKET }}
          B2_S3_ENDPOINT: ${{ secrets.B2_S3_ENDPOINT }}
          B2_BACKUP_KEY_ID: ${{ secrets.B2_BACKUP_KEY_ID }}
          B2_BACKUP_APP_KEY: ${{ secrets.B2_BACKUP_APP_KEY }}
          BUNDLE: ${{ steps.bundle.outputs.bundle }}
        run: |
          set -euo pipefail
          scripts/backup/upload.sh --file "$BUNDLE" --key "clint/secrets/$(basename "$BUNDLE")"

  notify-failure:
    needs: [export]
    if: failure()
    runs-on: ubuntu-latest
    permissions:
      issues: write
    steps:
      - uses: actions/github-script@v7
        with:
          script: |
            const label = 'secrets-break-glass-failure';
            const title = 'Secrets break-glass export failed';
            const body = `The weekly secrets break-glass export failed on run ${context.runId}. ` +
              `Without it, there is no fresh offline copy of secrets if Infisical is lost. ` +
              `See https://github.com/${context.repo.owner}/${context.repo.repo}/actions/runs/${context.runId}`;
            const existing = await github.rest.issues.listForRepo({
              owner: context.repo.owner, repo: context.repo.repo,
              state: 'open', labels: label });
            if (existing.data.length === 0) {
              await github.rest.issues.create({
                owner: context.repo.owner, repo: context.repo.repo,
                title, body, labels: [label] });
            }
```
