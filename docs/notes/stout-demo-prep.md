## Part 7 — Feature reference (for answering live)

Five Landscape lenses on **one shared, filtered dataset** (filters follow you between them): **Space home · Timeline · Bullseye · Heatmap · Catalysts.**

- **Space home** — the morning screen: next catalyst, pulse signals (P3 readouts / catalysts 90d / new analysis 7d / trial moves 30d), latest from Stout, what changed, recent files. The proof of "live."
- **Timeline** — every trial on one time axis, grouped Company→Asset→Trial; color=category, fill=confidence (filled=confirmed, hollow=projected); dashed line=today; auto-READ in plain English. Export → PPT.
- **Bullseye** — distance to center = closeness to market; rings=phase, wedges=grouping (Company/MoA/Indication/…); blue ring=intelligence attached, amber=recent activity. Switch GROUP BY to MoA = "which *science* is winning."
- **Heatmap** — crowding vs. white space; rows=buckets, cols=phases, dark=crowded, empty=open lane. The BD/licensing view.
- **Future Catalysts** — chronological watch list (this week → quarter); confirmed vs. projected; Export → Excel.
- **Primary Intelligence** — Stout's read, written in place on any trial/company/asset; SUMMARY (what's happening) + IMPLICATIONS (so-what); draft→publish→version; **History = word-level diffs**; drafts are Stout-only, client sees published. *This is where Stout's IP lives.*
- **Source Import** — paste NCT IDs / URL / text → AI builds companies/assets/trials/catalysts; provenance on every entry. The live AI feature.
- **Activity / Events** — daily CT.gov sync + typed, sourced change records (date slips, phase moves, status, protocol). Powers "what changed."
- **Materials** — deliverables filed against the entities they cover (Briefing / Conference report / Priority notice / Ad hoc), surface where the client looks + one library.
- **Whitelabel** — Agency (Stout) → Tenant (client) → Space (engagement); client's brand foreground, "Intelligence by Stout" in the header on every screen; Stout runs it all from one portal.
- **Security** — per-client walls + per-space explicit membership (one engagement never sees another's); encrypted daily backups across two clouds, restores rehearsed; AWS + Cloudflare. Clint runs all of it.

## Part 8 — Domain cheat-sheet

**Hierarchy:** Company → Asset (drug) → Trial (has an NCT number) → Marker (dated event = the dots). **Catalyst** = a future marker.

**Phases:** Preclinical → P1 (first-in-human, safety) → P2 (efficacy signal) → **P3 (pivotal, highest-stakes readouts)** → Filed → Approved → Launched → LOE (patent cliff).

**Marker colors:** green=data readouts (Topline/Interim/Full) · slate=trial milestones (Primary Completion/Start/End) · orange=regulatory (Filing/Submission/Acceptance) · blue=approval · violet=launch · amber=loss of exclusivity. **Fill:** filled=confirmed · hollow=projected · struck-through=no longer expected.

**Vocab:** Topline (first headline result — the market-mover) · Primary completion (data collection done; precedes topline) · Confirmed vs Projected (known vs estimated date) · MoA (how it works) · RoA (SC=injection, PO=oral) · Indication (disease) · NDA/BLA (US filings) · PDUFA (FDA decision date) · head-to-head (vs a competitor).

**Confusables:** Primary Completion ≠ Trial End · Interim→Topline→Full (peek→headline→complete) · Filing ≠ Acceptance (sent vs. agreed-to-review) · Approval ≠ Launch (allowed vs. selling) · LOE ≠ Generic Entry · trial Phase (maxes at P4) ≠ asset Development Status (→ Approved/Launched, per indication).
