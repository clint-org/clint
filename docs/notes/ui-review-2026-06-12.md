# UI Review, dev site, 2026-06-12

Screen-by-screen review of dev.clintapp.com (BI tenant, seeded space, Stout agency portal, super-admin). Combined findings from two passes: a design-critique pass and an interaction/consistency/a11y pass. Screenshots in /tmp/clint-ui-review/.

Status legend: [ ] open, [x] fixed, [-] won't fix / by design.

## P1: Functional or blocking

1. [x] **Landscape filter bar clips with no overflow affordance.** Fixed: the toolbar row wraps (`flex-wrap`) instead of scroll-clipping; verified on dev at 1512 and 1280. At 1512px the "Last CT.gov sync" caption and the To-period control are cut at the right edge; at 1280px the whole Period group disappears and the row ends mid-word at "PERIO". Affects Timeline, Bullseye, Heatmap, Future Catalysts. Needs wrap or a "More filters" collapse, and the sync caption moved out of the row. (04-timeline, 20-timeline-1280, 06-heatmap)
2. [ ] **Timeline marker labels collide at year zoom.** Overlapping date labels and glyphs wherever catalysts cluster: "JunJa'26'26", "Sep1t'26", "Mar Ap05 '25". Worst on the flagship Timeline; also on company/asset detail mini-timelines. Needs collision-aware placement (hide on overlap + hover reveal, or cluster badge). (04, 20, 23, 25)
3. [ ] **Space owner blocked from Import.** Navigating to /import as space owner + tenant owner bounces to Home with "Editor access required to import data." Either the importGuard role check is wrong or the implicit-role mapping is; verify intended behavior. (33-import)
4. [ ] **Marker date labels are 8px and fail AA contrast.** Orange #f97316 (~2.8:1) and blue #3b82f6 (~3.6:1) on white at 8px; slate-400 dates too. These carry real data. Brand guide calls AA a hard floor. (04)
5. [ ] **Slate-400 meaningful text fails AA on white.** PERIOD label, stat unit suffixes ("co", "assets", "trials", "cat/90d") at 10-11px in #94a3b8 (~2.9:1). Footer legend on Timeline: 9px slate-500 on slate-900 (~3.0:1) for "Actual/Projected/NLE". Audit which slate-400/500 uses carry meaning and bump them. (04, eval data)

## P2: Design and consistency

6. [ ] **Primary button color drifts across the app.** "Add primary intelligence" is brand blue; space settings "Save changes" is violet (a data color, Phase 4/launch); agency Branding "Save" is washed-out light teal that reads disabled; super-admin "Provision agency" is solid teal inside a navy shell. One primary treatment per brand context. (14, 26, 18, 19)
7. [ ] **Navigation IA inconsistencies.**
   - Engagement: rail groups it under INTELLIGENCE, header shows "Manage" + Companies/Assets/Trials tabs, and the current page is not in the visible tab set (URL is /manage/engagement). (15)
   - Settings pages have no sibling tabs; lateral movement among the 6 settings pages requires the hover rail. (16, 26)
   - Settings page title placement is inconsistent: Members/General put the title in the top bar, Audit log puts an h1 in the content area with an empty top bar. (16, 26, 28)
   - The five Landscape tabs are duplicated as the first five of 18 unlabeled rail icons. Decide whether the rail should repeat them. (snapshot of nav)
   - Trial detail introduces a fourth nav idiom: mono pipe-separated anchor links (PRIMARY INTELLIGENCE | TIMELINE | ...) with no affordance distinguishing them from static labels. (25)
8. [ ] **"Clear filters" rendered disabled on every list view even when no filters exist.** Catalysts, Events, Companies, Assets, Trials, Marker Types. Hide it until at least one filter is active. (08, 11, 12, 13, 24, 27)
9. [ ] **Future Catalysts: PROJECTED badge on every row carries no signal.** Badge only the exceptions (confirmed/actual), or de-emphasize the default. (08)
10. [ ] **Events table readability.** "Primary Completion Date (P..." truncates mid-word in nearly every row while ENTITY wraps trial codes badly ("SURPASS-" / "2"); give the title column priority, no-wrap trial codes. Sort+filter glyph pairs on every header add clutter; consider showing them on hover/active only. (11)
11. [ ] **Status chip colors don't differentiate.** Trials list: TERMINATED renders the same gray as COMPLETED; it should read as a negative state at a glance. (24)
12. [ ] **Heatmap left-panel idiom mismatch.** GROUP BY is five stacked full-width buttons while COUNT directly below is a segmented control; same kind of choice, two idioms. Bullseye has the same stacked GROUP BY. (06, 05)
13. [ ] **Heatmap labels.** Row labels ellipsize ("GIPR antagonist + GLP-1 a...") with no tooltip; "LNCH" and "APP" column headers are cryptic next to P1-P4. (06)
14. [ ] **Bullseye issues.** PH 2/PH 3 ring labels collide with asset dots; center reads "Filtered, 22 assets" when no filter is applied; STATS says "12 spokes" (jargon); Period filter relevance to this view unclear. (05)
15. [ ] **Spaces page ghost cell.** The cards grid uses gap-px + bg-slate-200 for hairline borders, so the empty third lg column renders as a solid gray rectangle. Fill empty cells white, or use a dashed "New space" ghost card, or cap the grid width. (02, 31; root cause confirmed in DOM)
16. [ ] **Super-admin agency rows: bare red trash as the only action.** One click from the most destructive tier-1 operation. Move delete into a kebab or detail view with type-to-confirm. Also "MAX TENANTS" and "TENANTS" adjacent columns; "Tenants 1/5" is tighter. (19)
17. [ ] **Members role column mixes treatments.** Self gets a static teal OWNER badge, others get a dropdown; same column, two visuals, no explanation. Also the badge is hard-coded teal on a blue-brand tenant; check role badges use brand or neutral tokens. (16)
18. [ ] **Agency Branding form.** Save button reads disabled (see 6); raw default "unknown@unknown.invalid" surfaced in the contact email field; right half of the page is empty where a live brand preview (color + logo applied to a mini header) belongs. (18)
19. [ ] **Default-host landing is the weakest screen.** Centered logo + tagline + one card floating in dead space; generic, doesn't sell or resemble the instrument inside. Consider a product-render backdrop or live timeline preview. "Go" button label is terse. (01)
20. [ ] **Command palette polish.** No backdrop scrim, the busy timeline competes with the panel; entity rows use plain color squares as icons (placeholder feel); no footer keyboard hints; the list cuts off mid-row at the bottom with no scroll fade. (21)
21. [ ] **Export inconsistencies.** Export lives top-right on Landscape pages but in-content top-left on Events/Companies/Assets/Trials; search box is right-aligned on Catalysts, left on Events. The heatmap Export menu items (Image PNG, Excel XLSX) lack the file-type icons Materials rows have. (07, 08, 11, 12)
22. [ ] **Detail mini-timelines clip phase bars flat at the left edge.** Bars that start before the visible window are cut with no fade or "starts earlier" affordance. (23, 25)
23. [ ] **Trial detail header repeats the name.** "ATTAIN-1  ATTAIN-1" (display name + raw name when identical). Suppress the duplicate. (25)

## What's working (keep, and use as the bar)

- Space Home: editorial hero (big date numeral + next catalyst), stat strip, NEXT 90 DAYS rail. The strongest screen; the utility pages should chase it.
- Mono tracked section labels (READ, STATS, LEGEND, NEXT 90 DAYS) as a consistent system signature.
- Catalysts time grouping (THIS WEEK, NEXT WEEK, months, quarters).
- Materials list: chip filters, file-type iconography, tight meta line.
- Admin shells (agency portal + super-admin) share one coherent pattern.
- Create space dialog: brand accent bar, educational copy, clear hierarchy.
- Branded 404, good Archived empty state, markers help page live-rendering marker types.
- Hover-expand dark rail with labeled groups and pin control.
