Engagement landing / what changed

Row composition options for the activity widget

The current row leads with the data source ("STOUT STRATEGY" or "CT.GOV"), then date, then NCT, then the actual change. Source has only two values and is the least useful axis. These eight variations explore what should anchor the row instead -- the change itself, the trial program, the asset/company, or a structured event tag.

Same five events render in every variation. Some need only component changes; others need the activity-feed RPC to join product / company. The orange flag below each variation says which.

V1

Baseline -- what is shipped today

Reference point. Source is the lead, in mono uppercase ("STOUT STRATEGY"); date and NCT follow on the same metadata line. Change summary lives below.

Color treatments

V1 structure, with color added semantically

Keep the row layout unchanged from baseline. Add color keyed to event meaning -- using the marker palette already established in the rest of the app. Each treatment increases color load by a step so you can pick the right intensity.

C1

Color the icon only -- minimal

Icon glyph picks up an event-type color; meta and summary stay slate. Lowest color load. Reads as a small accent that helps you scan event types without changing the row's typographic rhythm.

Inherit existing color taxonomy + strikethrough on the delta

Recommended

Color appears only where the event maps to an established taxonomy in the app. Phase transitions pick up the destination phase color; marker events pick up the marker's category color. Everything else stays slate -- so color always reinforces meaning the user already knows. The "from" value is struck through; the "to" value is bold and colored where the taxonomy applies.

Color rules phase_transitioned → destination phase color  ·  marker_added/updated/removed → marker's category color  ·  everything else → slate-700

Event-type color assignments (used in C1-C4)

Violet
phase_transitioned, marker_reclassified, sponsor_changed -- structural pivots
Amber
date_moved (delays), trial_withdrawn -- timeline / regulatory issues
Red
status terminated/withdrawn/suspended, arm_removed, marker_removed
Green
marker_added, projection_finalized, status completed -- data positives
Blue
enrollment_target_changed, intervention_changed, outcome_measure_changed, marker_updated -- design changes
Cyan
arm_added, eligibility_changed -- protocol additions (reserved)

Data shown / hidden across variations

Source
Two values only: ctgov (CT.gov) or analyst (the agency, e.g. Stout Strategy). Encoded as a 2-letter chip (CT / AN) in V2-V8 instead of the full agency name.
NCT
Hidden from row in V2-V8. Available on hover and used as the link target. Power users can still copy from the trial-detail page.
Program
Trial.name (e.g. KEYTRUDA-407, DESTINY-Breast09). Already joined in the RPC. The strongest identity anchor for a CI analyst.
Asset
Generic drug name. Not joined today -- requires extending get_activity_feed to join through to products. Used only in V6.
Company
Sponsor / lead developer. Not joined today -- same RPC change as Asset. Used only in V6.
Time
Switched from absolute ("May 5, 2026, 9:14:00 AM") to relative ("2h", "1d", "6d") in V2-V8. Absolute time stays available on hover.
